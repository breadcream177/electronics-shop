# Electronics Shop

PHP와 MySQL로 구현한 전자제품 쇼핑몰 개인 프로젝트입니다. 상품 탐색부터 장바구니, 주문, 결제 검증, 리뷰, Q&A, 관리자 주문 관리까지 전자상거래 서비스의 핵심 흐름을 직접 구현했습니다.

## 프로젝트 개요

- 개발 목적: 전자상거래 서비스의 주요 기능을 직접 설계하고 구현하며 PHP/MySQL 기반 웹 애플리케이션 흐름을 학습
- 개발 형태: 개인 프로젝트
- 주요 사용자: 전자제품을 검색하고 구매하려는 일반 사용자, 주문과 문의를 관리하는 관리자
- 저장소: https://github.com/breadcream177/electronics-shop

## 기술 스택

| 구분 | 기술 |
| --- | --- |
| Backend | PHP |
| Database | MySQL, mysqli |
| Frontend | HTML, CSS, JavaScript |
| Auth | 일반 로그인, 카카오 로그인, 구글 로그인 |
| Payment | PortOne/Iamport, KakaoPay |
| Local Environment | XAMPP, Apache |

## 주요 기능

### 사용자 기능

- 회원가입, 로그인, 로그아웃
- 카카오/구글 소셜 로그인
- 상품 목록 조회
- 상품명 검색
- 가격 범위 필터링
- 최신순, 가격순, 평점순 정렬
- 상품 상세 조회
- 장바구니 담기, 수량 변경, 삭제, 비우기
- 위시리스트 추가, 삭제, 장바구니 이동
- 주문서 작성

- 주문 내역 조회

- 비회원 문의 작성
- 마이페이지 조회

### 관리자 기능

- 관리자 접근 제어
- 주문 목록 조회

- 회원 목록 관리
- 리뷰 관리
- 비회원 문의 관리

## 구현 포인트

### 1. 상품 탐색 경험 구현

상품 목록 페이지에서 검색어, 최소/최대 가격, 정렬 조건을 조합해 상품을 조회할 수 있도록 구현했습니다. 리뷰 테이블을 집계한 평균 평점과 리뷰 수를 상품 목록에 함께 표시해 사용자가 상품을 비교하기 쉽도록 구성했습니다.

### 2. 장바구니와 주문 흐름 구현

사용자가 장바구니에 담은 상품을 주문서 작성 페이지에서 다시 조회하고, 서버에서 상품 가격과 총액을 재검증하도록 구현했습니다. 클라이언트에서 전달된 금액만 신뢰하지 않고 DB 기준 총액과 비교해 주문 데이터의 정합성을 확인했습니다.

### 3. 결제 검증 로직 구현

PortOne/Iamport API를 사용해 카카오페이 결제를 연동했습니다. 결제 완료 후 `merchant_uid`, 결제 금액, 결제 상태를 서버에서 다시 조회하고 DB 주문 정보와 비교해 검증한 뒤 주문 상태를 갱신합니다.

### 4. 중복 결제 처리 방지

이미 결제 완료된 주문에 대해 검증 페이지가 새로고침되거나 재호출되더라도 중복 처리되지 않도록 주문의 결제 상태를 확인하는 로직을 추가했습니다.

### 5. 관리자 주문 관리

관리자 페이지에서 주문 목록을 조회하고, 결제 완료/상품 준비/배송 중/배송 완료/취소 등 주문 상태를 변경할 수 있도록 구현했습니다.

## 실행 방법

### 1. 프로젝트 위치

XAMPP의 Apache 루트 아래에 프로젝트를 배치합니다.

```text
C:\xampp\htdocs\electronics_shop
```

브라우저에서 아래 주소로 접속합니다.

```text
http://localhost/electronics_shop/
```

### 2. 데이터베이스 설정

`DB/db.php`에서 MySQL 연결 정보를 설정합니다.

```php
$host = "localhost";
$user = "root";
$pass = "";
$db = "electronics_shop";
```

현재 저장소에는 DB 덤프 파일이 포함되어 있지 않으므로, 실행 환경에서는 `electronics_shop` 데이터베이스와 필요한 테이블을 별도로 생성해야 합니다.

주요 테이블 예시:

- `users`
- `products`
- `cart`
- `wishlist`
- `orders`
- `order_items`
- `reviews`
- `qna`
- `guest_inquiries`

### 3. 결제/소셜 로그인 설정

결제와 소셜 로그인은 외부 서비스 키가 필요합니다. 실제 운영 키는 저장소에 올리지 않고 로컬 설정 파일에서 관리하는 것을 권장합니다.

- `config_portone_v1.php`: PortOne/Iamport REST API 키 설정
- `kakao/kakao_config.php`: 카카오 로그인 설정
- `google/google_config.php`: 구글 로그인 설정
